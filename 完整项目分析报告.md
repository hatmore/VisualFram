## 项目概览

VisualFram2D 是一个**可视化任务流编程平台**，专门用于**实时视觉处理和深度学习推理**。该项目支持用户通过拖拽节点的方式构建复杂的图像处理流水线，无需编写代码即可实现自定义的视觉分析工作流。

### 主要功能

- ✅ **可视化节点编辑**: 基于QtNodes的任务流编程
- ✅ **深度学习推理**: 集成TensorRT、OpenCV、YOLOv11等
- ✅ **实时视频处理**: 支持摄像头、视频文件的实时流处理
- ✅ **插件系统**: 模块化DLL插件架构，易于扩展
- ✅ **姿态识别**: YOLOv11 Pose + PoseC3D 3D识别
- ✅ **外部通信**: 支持设备I/O和网络通信
- ✅ **自动化流程**: SOP报告、参数配置、持久化保存

### 应用场景

- 工业机器视觉检测
- 人体姿态分析
- 目标检测和实例分割
- 自动化生产线监控
- AI视频分析平台

### 核心设计原则

1. **模块化**: 通过插件DLL实现功能解耦

## 核心模块详解

### 1. AutoRunOperation - 自动运行模块

**位置**: `MianWindows/MianWindows/AutoRunOperation/`

管理后台任务的执行线程：

| 线程类 | 功能 |
|-------|------|
| **AutoGetAndRanderVideoStream** | 获取视频流并渲染（相机或视频文件） |
| **AutoSignalTriggerFlowsExeThread** | 处理信号触发的流程执行 |
| **AutoFlowsLogInfoShowThread** | 显示流程执行日志 |
| **AutoCommunicationDeviceWriteThread** | 外部设备通信写入 |
| **AutoFlowsExeThread** | 主流程执行线程 |
| **AutoSOPReportShowThread** | SOP报告显示 |

### 2. NodeEditor - 节点编辑器

**位置**: `Plugin/*/*/NodeEditor/` 和 `MianWindows/MianWindows/NodeEditor/`

- **BaseNodeModel.h**: 节点模型基类
- **BaseNodeData.h**: 节点数据包装
- **NodeEmbedWidget.h**: 节点嵌入小部件
- **OverrideDataFlowGraphModel.h**: 图模型（主应用）

数据流图模型负责：
- 节点连接管理

## 并发和性能

### 多线程架构

```
Main Thread (UI)
    │
    ├─► AutoGetAndRanderVideoStream (获取帧)
    │
    ├─► AutoFlowsExeThread (执行节点流)
    │
    ├─► AutoSignalTriggerFlowsExeThread (信号触发)
    │
    ├─► AutoFlowsLogInfoShowThread (日志显示)
    │
    ├─► AutoCommunicationDeviceWriteThread (I/O)
    │
    └─► AutoSOPReportShowThread (报告生成)
```

### 数据同步机制

- **信号槽**: UI线程与工作线程通信
- **BlockingQueuedConnection**: 异步调用后同步
- **concurrent_queue**: 线程安全数据队列
- **互斥锁**: 保护共享资源

### 性能优化

1. **GPU加速**: TensorRT推理在GPU上执行
2. **流水线处理**: 异步获取和处理分离
3. **线程池**: 复用线程对象
4. **数据缓冲**: 防止内存频繁分配释放
